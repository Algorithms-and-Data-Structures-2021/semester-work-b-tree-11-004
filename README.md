_Краткое описание семестрового проекта:_

-  **B-tree**
-  B-tree - это сбалансированное, сильно ветвистое дерево поиска.
-  B-tree хорошо подходит для систем хранения, которые читают и записывают относительно большие блоки данных, такие как диски. Обычно используется в базах данных и файловых системах.
-  **Основные свойства B-tree**:

   Пусть m - минимальная степень дерева. Тогда каждый узел, кроме корневого, должен иметь, как минимум m – 1, и не более 2m – 1 ключей.
   
   Ключи в узле хранятся в неубывающем порядке.
    
   Если узел не является листом и содержит n ключей (K<sub>1</sub>...K<sub>n</sub>), то он имеет n+1 потомков.
   При этом:
     - Первый потомок и все его потомки содержат ключи из интервала (-∞; K<sub>1</sub>);
     - Для 2≤i≤n, i-й потомок и все его потомки содержат ключи из интервала (K<sub>i-1</sub>; K<sub>i</sub>);
     - (n+1)-й потомок и все его потомки содержат ключи из интервала (K<sub>n</sub>; +∞).

   
   Все листья B-tree должны быть расположены на одной высоте, которая и является высотой дерева. Высота B-tree с n ≥ 1 узлами и минимальной степенью m ≥ 2 не превышает log<sub>m</sub>(n+1). 
   
   Над B-tree можно выполнять операции поиска, вставки и удаления.

- **Теоретическая сложность операций:** 
   - **Операция поиска** выполняется за время O(m log<sub>m</sub> n).
   - **Операция вставки** выполняется за время O(m log<sub>m</sub> n).
   - **Операция удаления** выполняется за время O(m log<sub>m</sub> n).

```shell

```

- `<input>` - входной файл с набором данных в формате CSV;
- `<output>` - выходной файл с результатами контрольного теста;
- `--trials` - количество прогонов на наборе данных и т.д.

Добавление 10000 случайных элементов в структуру данных (повторить операцию 50 раз и вычислить среднее время работы и
потребляемую память):

```
./add_benchmark.exe ../dataset/data/add/10000.csv metrics.txt --trials 50
``` 

где `<input> = ../dataset/data/add/10000.csv` и `<output> = metrics.txt`.

**Примечание**. Файл с метриками не обязателен, можете выводить данные в стандартный поток вывода (т.е. консоль).
